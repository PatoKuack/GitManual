git init: arranca un repositorio.
git add ___.txt: agrega el archivo o sus cambios al repositorio (en la memoria RAM). se prepara para ingresarse a la base de datos.
git add .: agrega todos los cambios que se hayan hecho a los archivos de la carpeta en donde se esta pocicionado.
git rm --cached __.txt: elimina el archivo del repositorio antes de que ingrese a la base de datos.
git commit: envia los ultimos cambios del archivo a la base de datos del sistema de control de versiones para controlar los cambios que se le hayan hecho. *Cada que se agregan (add) archivos se tiene que enviar los cambios a la base de datos (commit).
git commit -m "mensage": el guión y la eme son para indicar que se le agregará un mensaje para poder verlo en el futuro como una referencia del cambio.
git status: muestra el estatus de los cambios hechos, se usa para verificar los cambios.
git show: muestra todos los cambios historicos hechos, cuando y quien hizo los cambios.
git log ___.txt: muestra el historial completo del archivo.
git push: permite enviar a otro repositorio remoto lo que se esta haciendo, tambien puede traer lo de otros repositorios remotos.
git config: muestra una lista de todas las configuraciones que tiene git y como funcionan.
git config --list: muestra la configuracion por defecto del git y las que le faltan.
git config --list --show-origin: muestra las configuraciones guardadas.
git config --global user.name "NAME": modifica la configuracion de los usuarios globales de git colocando un nombre.
git config --global user.email "E-MAIL": lo mismo pero coloca un e-mal.
